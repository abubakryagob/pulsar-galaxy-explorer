== BACKEND: DATA PIPELINE (Node.js + Express + SQLite) ==

Create scripts/ingest.js — a one-time data ingestion script:

1. Parse atnf_raw.csv using the 'csv-parse' npm package
2. Parse magnetars_raw.txt (tab or space separated, handle manually)
3. Classify each object into one of these classes using this logic:

   MAGNETAR:        Type contains 'AXP' or 'SGR', OR name matches McGill list
   RECYCLED_MSP:    P0 < 0.03s AND Binary parameter is not null
   FAST_MSP:        P0 < 0.03s AND Binary is null
   RRAT:            Type contains 'RRAT'
   ISOLATED_NS:     Type contains 'XINS'
   CCO:             Assoc contains 'SNR' AND no S1400 flux AND no P1
   BINARY_PULSAR:   Binary is not null AND P0 >= 0.03s
   CANONICAL:       everything else

4. For each object, compute if not already present:
   - BSurf = 3.2e19 * sqrt(P0 * P1)  [Gauss]  (skip if P1 is null)
   - Age = P0 / (2 * P1)  [years]  (skip if P1 is null)
   - Edot = 4 * pi^2 * I * P1 / P0^3  where I=1e45 g/cm2  [erg/s]
   - audio_freq: map P0 to audio range 40-2000 Hz using:
     freq = 440 / P0  (cap at 2000, floor at 40)
   - is_msp: boolean, true if P0 < 0.03
   - has_binary: boolean

5. Store everything in SQLite table 'neutron_stars' with columns:
   id, jname, bname, ra, dec, gl, gb, 
   x_kpc, y_kpc, z_kpc, dist_kpc,
   p0, p1, dm, age_yr, bsurf_g, edot_ergs,
   s1400, w50, binary_model, assoc, nglitch,
   ns_class, audio_freq, is_msp, has_binary,
   discovery_date, source (atnf or mcgill),
   created_at

6. For magnetars from McGill catalogue, add additional columns:
   last_outburst_year, outburst_count, spectral_type

Create API endpoints:

GET /api/stars
  Returns all objects with: id, jname, gl, gb, x_kpc, y_kpc, z_kpc, 
  ns_class, p0, bsurf_g, is_msp, dist_kpc, audio_freq
  (lightweight — only what Three.js needs to render)
  
GET /api/stars/:jname
  Returns ALL columns for a single object (full detail panel)

GET /api/stars/class/:class
  Filter by ns_class

GET /api/stats
  Returns: count by class, min/max P0, min/max BSurf, 
  total objects, last updated timestamp

GET /api/ppdot
  Returns: id, jname, p0, p1, bsurf_g, ns_class, dist_kpc
  (for P-Pdot diagram — only objects where both P0 and P1 exist)

Also create GET /api/refresh that re-runs the ingestion 
(so you can update data without restarting the server).
